# Text-First PDF Pipeline - Реализовано ✅

## Описание

Новый пайплайн извлечения данных из PDF спецификаций с приоритетом на текстовый слой.

**Старый подход:** PDF → Картинка (4x zoom) → LLM Vision API → JSON → Excel  
**Новый подход:** PDF → pdfplumber (текст) → Таблица → Excel (с fallback на Vision)

## Преимущества

- ⚡ **Мгновенная работа** для PDF с текстовым слоем (без API-вызовов)
- 💰 **Бесплатно** для текстовых PDF (0 руб за конвертацию)
- 🌐 **Работает без интернета** для текстовых PDF
- 🔄 **Fallback на Vision** для сканов и страниц без текста
- 📄 **Смешанные PDF** - автоматически определяет режим для каждой страницы

## Что было исправлено в проекте

После сравнения результата text-first с эталоном (vision) были внесены следующие исправления:

### 1. Маппинг колонок по строке нумерации (1–9)

**Проблема:** pdfplumber возвращает 16–22 сырых колонки из-за объединённых ячеек; старый код просто удалял «пустые» колонки, из-за чего данные сдвигались и попадали не в те столбцы.

**Решение:** Поиск в таблице строки нумерации колонок (`1 2 3 4 5 6 7 8 9`) и использование её как карты: какой сырой индекс → какая логическая колонка (1–9). Маппинг свой для каждой страницы (разные макеты: 16, 18, 22 колонки).

### 2. Перекодировка CP1251 → UTF-8

**Проблема:** Часть текста в PDF хранится с битой кодировкой (CP1251 байты прочитаны как Latin1): «Ñåðèÿ», «ÌÇÂ», «VÒð.700», «Òðóáîïðîâîä».

**Решение:** Функция `fix_encoding()`: `text.encode('latin1').decode('cp1251')` с проверкой результата. Применяется к ячейкам, где есть подозрительные символы (0xC0–0xFF).

### 3. Заголовки из PDF

**Проблема:** На страницах 4–5 другой макет («Поз.», «Код продукции», «Поставщик», «Кол-во», «Масса 1 ед., кг»). Подставлялся один стандартный заголовок.

**Решение:** Из первой строки таблицы извлекается заголовок по тому же маппингу колонок; для страниц с «Поз.» / «Код продукции» он используется вместо стандартного. Вертикальная надпись «я и ц и з о П» заменяется на «Позиция».

### 4. Фильтрация штампа и рамки

**Проблема:** В данные попадали строки штампа (Изм., Подп., Разработал, номер документа, вертикальные надписи «Взам. инв. №» и т.п.), в т.ч. с битой кодировкой.

**Решение:** Расширен список ключевых слов штампа (нормальные + битая кодировка, названия проектов). Фильтрация вертикальных надписей (много `\n` между короткими фрагментами).

### 5. Пост-обработка ячеек

- **Слипшиеся ячейки:** например «3 Трап вертикальный...» в одной колонке → разделение на позицию «3» и наименование «Трап вертикальный...»; «шт 4» в колонке кода → единица «шт» и количество «4».
- **Склейка тип + код:** если в колонке «Тип» значение вроде «SML», а в «Код» — только число («10101»), они склеиваются в «SML 10101» в колонке типа.
- **«м/м3»:** разделение на единицу «м/м» и количество «3».
- **Нормализация позиции:** «-» в колонке «Позиция» заменяется на пустую строку (как в эталоне).
- **Пробелы вокруг кавычек:** «ЛЗТА " Маршал"» → «ЛЗТА "Маршал"».

### 6. Дефисы-переносы в заголовках

**Проблема:** В PDF заголовки с переносами слов: «Единица измере- ния», «Коли- чество».

**Решение:** В извлечённых заголовках удаляются дефисы-переносы (`-\s+`), лишние пробелы схлопываются.

Итог: **~90,6% совпадения ячеек с эталоном**, лист 4 — **100%**, листы 1 и 5 — **97–98%**. Оставшиеся расхождения в основном из-за другого порядка строк на листах 2 и 3 (порядок в PDF сохранён; в эталоне он был переупорядочен вручную).

## Известные ограничения

1. **Порядок строк на листах 2 и 3** — в PDF шаровые краны идут перед пожарным оборудованием; в эталоне порядок был изменён вручную. Text-first сохраняет порядок из PDF.

2. **Многострочные значения** — в эталоне в ячейках иногда сохранён `\n` (например, два ГОСТ в одной ячейке); после извлечения текста переносы заменяются на пробел.

## Структура кода

### Модуль `backend/pdf_text_extractor.py`

- `has_text_layer(pdf_path)` — проверка наличия текстового слоя по страницам
- `extract_table_from_text(page)` — извлечение таблицы; возвращает `(rows, pdf_header)`
  - `find_column_mapping(table)` — поиск строки нумерации (1–9) и построение маппинга сырых колонок → 9 логических
  - `_extract_pdf_header()` — извлечение заголовка из первой строки таблицы
  - `_fix_merged_cells(row)` — разделение слипшихся ячеек, склейка SML+код, нормализация позиции и кавычек
- `normalize_table_to_9cols(raw_rows, page_num, page_header)` — добавление заголовка, очистка пустых строк
- `fix_encoding(text)` / `fix_row_encoding(row)` — перекодировка CP1251→UTF-8 для битых строк
- `detect_sheet_name()` — определение имени листа по кодам систем (В1-3, Т3 Т4, К1)
- `detect_encoding_quality()` — проверка «битости» текста (для справки)

## Логика работы

```python
for каждой страницы в PDF:
    if has_text_layer(страница) and keywords_found:
        # TEXT MODE (быстро, бесплатно)
        таблица = extract_table_from_text(страница)
        if таблица успешно извлечена:
            normalized_data = normalize_table_to_9cols(таблица)
            добавить в результат
            continue
    
    # VISION MODE (fallback)
    картинка = pdf_to_image(страница)
    данные = extract_via_LLM_vision(картинка)
    добавить в результат
```
