# 🔄 Как обновить ваш проект

## Быстрое решение (2 минуты)

### Вариант 1: Замена одного файла

```bash
# 1. Остановите сервер (Ctrl+C если запущен)

# 2. Перейдите в ваш проект
cd ~/projects/extraction_specific_PDF/spec-converterv2/backend

# 3. Сделайте бэкап старого файла
cp app.py app.py.old

# 4. Скачайте новый app.py из обновлённого архива
# (или скопируйте из spec-converter.zip → backend/app.py)

# 5. Перезапустите сервер
cd ..
./start.sh
```

### Вариант 2: Полная переустановка

```bash
# 1. Остановите сервер (Ctrl+C)

# 2. Сохраните ваш config.py
cd ~/projects/extraction_specific_PDF/spec-converterv2/backend
cp config.py ~/config.py.backup

# 3. Распакуйте новый spec-converter.zip в другую папку

# 4. Скопируйте ваш config.py обратно
cp ~/config.py.backup ./новая_папка/spec-converter/backend/config.py

# 5. Запустите из новой папки
cd ./новая_папка/spec-converter
./start.sh
```

## Проверка что обновление работает

После запуска вы должны увидеть:

```
======================================================================
🚀 Конвертер спецификаций (ИСПРАВЛЕННАЯ ВЕРСИЯ)
======================================================================
✅ Провайдер: openrouter
✅ Модель: anthropic/claude-sonnet-4
```

Обратите внимание на **(ИСПРАВЛЕННАЯ ВЕРСИЯ)** - это означает что используется новый код!

## Что изменилось

1. ✅ **Заголовки** - первая строка каждого листа теперь содержит названия колонок
2. ✅ **Несколько листов** - данные разбиты по секциям (В1-3, Т3-Т4, К1 и т.д.)
3. ✅ **Точность** - AI лучше распознаёт буквы (В вместо ВТ, У вместо Ч)
4. ✅ **Полнота** - все строки из PDF попадают в Excel

## Тест

Загрузите ваш PDF "Спецификация 2273-1.1-ВК2 Офисы.pdf" и проверьте результат:

### ✅ Правильный результат:
- 3-5 листов (по секциям)
- Первая строка: ["Позиция", "Наименование", "Тип", ...]
- Правильные позиции: "В1-3", "Т3-Т4", "К1"
- ~70-80 строк данных всего

### ❌ Старый результат (если не обновили):
- 1 лист "Спецификация"  
- Нет заголовка
- Ошибки: "ВТ-3", "ЧВТ" вместо "В1-3", "УВ1"
- Меньше строк
