# Диагностика: НЕправильный парсинг спецификаций ОВ.СО

> **Дата:** 2026-03-17
> **Файлы-образцы:** НЕправильный_парсинг_1.pdf (2273-1.2-ОВ.СО листы 5-7), НЕправильный_парсинг_2.pdf (2273-1.2-ОВ.СО листы 1-4)
> **Затронутый модуль:** `services/spec-converterv2/backend/pdf_text_extractor.py`

---

## 1. Корневая причина

**`find_column_mapping()` не находит строку-нумератор (1 2 3 4 5 6 7 8 9) → код уходит в `_extract_fallback()` → данные разбиваются неправильно.**

PDF-файлы `2273-1.2-ОВ.СО` (ОВ — отопление/вентиляция) имеют другую структуру таблицы, чем ВК-спецификации, на которых разрабатывался пайплайн:

- **Правильно парсятся (ВК):** PDF содержит строку `1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 | 9` — `find_column_mapping()` строит точный маппинг сырых колонок → 9 логических.
- **НЕправильно парсятся (ОВ):** PDF НЕ содержит строку-нумератор (или содержит её в другом формате) — `find_column_mapping()` возвращает `None` → `_extract_fallback()`.

## 2. Что делает `_extract_fallback()` и почему он ломает данные

```python
def _extract_fallback(table):
    none_cols = _find_always_none_columns(table)  # убрать пустые колонки
    for row in table:
        clean = [cell for i, cell if i not in none_cols]
        if len(clean) > 9: clean = clean[:9]      # ОБРЕЗАТЬ до 9
        elif len(clean) < 9: clean += [''] * ...   # ДОБИТЬ до 9
```

Проблема: pdfplumber возвращает, например, 12-15 сырых колонок (из-за вложенных линий в PDF). Fallback просто берёт первые 9 непустых — и данные сдвигаются. Колонки «Ед.изм», «Кол-во», «Масса» оказываются за пределами первых 9 и обрезаются.

## 3. Конкретные симптомы (доказано анализом xlsx)

| Симптом | Пример из НЕправильного xlsx | Правильное значение |
|---------|------------------------------|---------------------|
| **Позиции пусты** | `(пусто) \| Биметаллический радиатор...` | `54. \| Биметаллический радиатор...` |
| **Данные в чужих колонках** | col5=`BVR-FR`, col7=`065B8304RG` | col3=`BVR-FR`, col4=`065B8304RG` |
| **Ед.изм/Кол-во/Масса пусты** | все три колонки пусты | `шт. \| 70 \| 0.37` |
| **Многострочный разрыв** | `D =15` + `У` (две строки) | `DУ=15` (одна ячейка) |
| **ГОСТ в чужой колонке** | col3=`Регистр из гладких труб...`, col5=`ГОСТ 10704-91` | col2=`Регистр...`, col3=`ГОСТ 10704-91` |

## 4. План исправлений

### 4.1. Улучшить `_extract_fallback()` — маппинг по заголовку (ГЛАВНЫЙ FIX)

Когда нумератор не найден, искать строку-заголовок ("Позиция", "Наименование", "Тип, марка"...) и строить маппинг по ней.

**Файл:** `pdf_text_extractor.py`
**Функция:** новая `_find_header_mapping()` + изменение `_extract_fallback()`

```python
def _find_header_mapping(table: List[List]) -> Optional[Dict[int, int]]:
    """
    Fallback-маппинг: ищет строку-заголовок таблицы по ключевым словам
    и строит маппинг raw_col → logical_col (1-9).
    
    Ключевые слова для каждой логической колонки:
      1: "позиция", "поз."
      2: "наименование"
      3: "тип", "марка", "обозначение"
      4: "код"
      5: "завод", "изготовитель", "поставщик"
      6: "единица", "ед.", "измер"
      7: "количество", "кол-во", "кол."
      8: "масса"
      9: "примечание"
    """
    HEADER_PATTERNS = {
        1: ["позиция", "поз."],
        2: ["наименование"],
        3: ["тип", "марка", "обозначение"],
        4: ["код"],
        5: ["завод", "изготовитель", "поставщик"],
        6: ["единиц", "ед.", "измер"],
        7: ["количеств", "кол-во", "кол."],
        8: ["масса"],
        9: ["примечан"],
    }

    for row_idx, row in enumerate(table[:5]):
        if row is None:
            continue

        # Проверяем: содержит ли строка хотя бы 4 ключевых слова заголовка
        row_texts = []
        for cell in row:
            val = str(cell).strip().lower() if cell else ""
            # Исправляем кодировку для проверки
            if needs_encoding_fix(val):
                val = fix_encoding(val).lower()
            row_texts.append(val)

        matches = {}  # logical_col → raw_col_idx
        for col_idx, text in enumerate(row_texts):
            if not text:
                continue
            for logical_num, keywords in HEADER_PATTERNS.items():
                if logical_num in matches:
                    continue  # уже нашли эту логическую колонку
                if any(kw in text for kw in keywords):
                    matches[logical_num] = col_idx
                    break

        # Если нашли >= 5 колонок — это заголовок
        if len(matches) >= 5:
            return matches

    return None
```

**Изменение в `extract_table_from_text()`:**

```python
    col_map = find_column_mapping(largest_table)

    if not col_map:
        # Fallback 1: маппинг по заголовку
        header_map = _find_header_mapping(largest_table)
        if header_map:
            col_map = {raw_idx: logical_num 
                       for logical_num, raw_idx in header_map.items()}
            # Продолжаем ОСНОВНОЙ алгоритм с этим маппингом
            # (не уходим в _extract_fallback)
        else:
            # Fallback 2: старый fallback (крайний случай)
            return _extract_fallback(largest_table), None
```

**Риск:** Минимальный. Для PDF с нумератором ничего не меняется. Для PDF без нумератора — вместо слепого обрезания теперь используется маппинг по заголовку.

### 4.2. Постобработка: склейка разрыва `D\nУ` → `DУ`

pdfplumber извлекает подстрочный индекс «У» отдельной строкой.

**Файл:** `pdf_text_extractor.py`
**Где:** в цикле `extract_table_from_text()`, после `val = " ".join(val.split('\n')).strip()`

```python
# Уже есть:
val = " ".join(val.split('\n')).strip()

# ДОБАВИТЬ после:
# Исправить разрыв подстрочных индексов (DУ, РУ, DН)
val = re.sub(r'\bD\s+У\b', 'DУ', val)
val = re.sub(r'\bР\s+У\b', 'РУ', val)  
val = re.sub(r'\bD\s+Н\b', 'DН', val)
val = re.sub(r'\bР\s+Н\b', 'РН', val)
# Также: "Ø" может разорваться
val = re.sub(r'Ш\s*=\s*', 'Ø = ', val)  # "Ш = 28" → "Ø = 28"
```

**Риск:** Нулевой. Паттерн `D У` (латинская D + пробел + кириллическая У) не встречается в нормальном тексте.

### 4.3. Склейка осиротевших строк-продолжений

Когда pdfplumber разрывает многострочную ячейку на отдельные строки таблицы, нужно склеить их обратно.

**Файл:** `pdf_text_extractor.py`
**Где:** новая функция `_merge_continuation_rows()`, вызывается после основного цикла фильтрации

```python
def _merge_continuation_rows(rows: List[List[str]]) -> List[List[str]]:
    """
    Склеивает строки-продолжения с предыдущей строкой.
    
    Строка считается продолжением если:
    - Позиция (col[0]) пуста
    - Только 1-2 колонки заполнены (обычно col[1] или col[2])
    - Заполненные колонки НЕ содержат числовых данных в числовых полях
    - Текст короткий (< 60 символов) и не начинается с "-" (не подпозиция)
    """
    if len(rows) < 2:
        return rows

    merged = [rows[0]]

    for row in rows[1:]:
        pos = row[0].strip()
        filled_cols = [(i, c) for i, c in enumerate(row) if c.strip()]
        
        # Это самостоятельная строка если:
        # - Есть позиция (число с точкой)
        # - Или текст начинается с "-" (подпозиция типа "-n=4 секции")
        # - Или заполнены >= 3 колонки
        # - Или в колонках 5-7 есть данные (ед.изм, кол-во, масса)
        has_position = bool(re.match(r'^\d+\.?', pos))
        starts_with_dash = row[1].strip().startswith('-') if row[1].strip() else False
        has_numeric_data = any(row[i].strip() for i in [5, 6, 7] if i < len(row))
        
        is_continuation = (
            not pos 
            and not has_position 
            and not starts_with_dash
            and not has_numeric_data
            and len(filled_cols) <= 2
            and len(merged) > 0
        )

        if is_continuation:
            # Склеиваем с предыдущей строкой
            prev = merged[-1]
            for i, cell in enumerate(row):
                if cell.strip():
                    if prev[i].strip():
                        prev[i] = prev[i].rstrip() + " " + cell.strip()
                    else:
                        prev[i] = cell.strip()
        else:
            merged.append(row)

    return merged
```

**Вызов:** в `extract_table_from_text()`, перед `return filtered_rows, pdf_header`:

```python
    # Склеиваем строки-продолжения
    filtered_rows = _merge_continuation_rows(filtered_rows)
    
    return filtered_rows, pdf_header
```

**Риск:** Средний. Нужно проверить на правильно парсящихся PDF. Критерий `is_continuation` достаточно строгий: строка должна быть без позиции, без «-», без числовых данных в правых колонках, и содержать ≤ 2 заполненных ячейки.

### 4.4. Извлечение позиции из текста наименования

Когда fallback кладёт «54. Биметаллический радиатор» целиком в col[1], позиция остаётся пустой.

**Файл:** `pdf_text_extractor.py`  
**Где:** добавить в `_fix_merged_cells()` в начало

```python
def _fix_merged_cells(row: List[str]) -> List[str]:
    # --- НОВЫЙ БЛОК: извлечение позиции из наименования ---
    # Если позиция пуста, а наименование начинается с "54." или "54.1."
    if not row[0].strip() and row[1].strip():
        match = re.match(r'^(\d+(?:\.\d+)?\.?)\s+(.+)$', row[1].strip(), re.DOTALL)
        if match:
            candidate_pos = match.group(1)
            # Позиция: 1-3 цифры с возможными точками (54., 54.1., 78.)
            if re.match(r'^\d{1,3}(?:\.\d{1,2})?\.?$', candidate_pos):
                row[0] = candidate_pos
                row[1] = match.group(2).strip()
    
    # ... остальной код _fix_merged_cells без изменений ...
```

**Риск:** Низкий. Работает только когда col[0] пуста. Паттерн `^\d{1,3}(\.\d{1,2})?\.?` матчит только типичные позиции спецификаций.

## 6. Проверка: что НЕ должно сломаться

После каждого исправления проверить на 4 образцах:
- `правильный_парсинг_1.pdf` (ОВ2.С, 16 страниц) — должен парситься так же
- `правильный_парсинг_2.pdf` (ВК2, 5 страниц) — должен парситься так же  
- `НЕправильный_парсинг_1.pdf` (ОВ.СО листы 5-7) — должен парситься правильно
- `НЕправильный_парсинг_2.pdf` (ОВ.СО листы 1-4) — должен парситься правильно

Критерии правильности:
- Позиции заполнены (54., 54.1., 55., 62., 68. и т.д.)
- Данные в правильных колонках (Тип/марка в col3, Код в col4, Завод в col5)
- Ед.изм, Кол-во, Масса не пусты там, где в PDF они есть
- Нет разрыва `D\nУ` — всё в одной ячейке как `DУ=15`
- Многострочные ячейки склеены (нет осиротевших строк «МПа», «нерж.»)
